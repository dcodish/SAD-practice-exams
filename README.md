## מה יש כאן

שלושה מבחנים מלאים, כל אחד בתיקייה משלו. לכל מבחן: קובץ המבחן (Word) + עמוד פתרון לדוגמה עם כל התרשימים הפתורים.

| מבחן | הסיפור (דומיין) | האובייקט המרכזי |
|---|---|---|
| **מבחן 1**, `2026-2-sample-1/` | ניהול השכרת דירות לסטודנטים ("קמפוס בית") | חוזה שכירות (`RentalAgreement`) |
| **מבחן 2**, `2026-2-sample-2/` | רשת מסעדות והזמנות שולחן ("שולחן פנוי") | הזמנה (`Reservation`) |
| **מבחן 3**, `2026-2-sample-3/` | רשת ספריות והשאלות ("ספר פתוח") | השאלה (`Loan`) |

בכל תיקייה:

```
<מבחן>/
  <מבחן>-exam.docx            ← קובץ המבחן, הורידו והדפיסו/כתבו עליו
  <מבחן>-solution/
    <מבחן>-solution.html      ← ★ עמוד הפתרון לדוגמה (פתחו בדפדפן)
    diagrams/                 ← ארבעת התרשימים הפתורים, אינטראקטיביים
      bpmn-solution.html, usecase-solution.html, class-solution.html, state-solution.html
```

## איך משתמשים

1. **פתרו קודם בעצמכם.** הורידו את `...-exam.docx`, הדפיסו או ענו במחברת, בדיוק כמו במבחן.
2. **אחר כך בדקו מול הפתרון.** פִתחו את `...-solution.html` בדפדפן, הוא כולל את כל התרשימים הפתורים ואת התשובות לשאלות התיאוריה והנימוק, עם הסבר לכל אחת.
3. **חִקרו את התרשימים.** בעמוד הפתרון יש קישור "פתחו את התרשים המלא" לכל תרשים, הם **אינטראקטיביים**: לחיצה על רכיב (Use Case, מצב, מחלקה) פותחת חלון עם פירוט. עובד גם ללא אינטרנט.
4. **שאלו.** כל שאלה, אי-בהירות או טעות שמצאתם, ראו למטה.

## איך לשאול שאלה או לדווח על טעות

יש שתי דרכים (שתיהן דורשות **חשבון GitHub חינמי**, [להרשמה](https://github.com/signup)):

- **הכי פשוט, ישירות בעמוד הפתרון.** בתחתית כל עמוד `...-solution.html` יש תיבת תגובות ("שאלות ותגובות"). כתבו שם, וה תגובה נשמרת אוטומטית ב-Discussions.
- **או דרך לשונית [Discussions](https://github.com/dcodish/SAD-practice-exams/discussions).** פתחתי דיון לכל מבחן, כִּתבו בדיון של המבחן הרלוונטי, או פתחו דיון חדש ("New discussion", קטגוריה Q&A).

בכל מקרה, **ציינו את מספר השאלה** שאתם שואלים עליה (למשל "שאלה 7.3"), כדי שנוכל לענות בדיוק. נענה ונתקן שם, וכך כל הסטודנטים נהנים מהתשובה.
